# Songwriter Player

Плагин для Obsidian: продвинутый аудиоплеер в правой боковой панели для сочинения песен.

Встроенное аудио в Obsidian выгружается при прокрутке заметки, и воспроизведение обрывается. Songwriter держит плеер в отдельной панели: скролль текст сколько угодно — музыка играет. Плеер переживает даже закрытие панели (звук живёт в плагине, а не в интерфейсе).

## Возможности

- **Панель в правом сайдбаре** — открывается кнопкой на ленте (нота) или командой.
- **Подхват из заметки** — плеер сам берёт аудио из открытой заметки (embed `![[song.mp3]]` или ссылка `[[song.mp3]]`). Если музыка уже играет, предлагает переключиться плашкой, не прерывая. Режим настраивается: гибрид / всегда / вручную.
- **Волна трека**:
  - одиночный клик — играть с этого места (клик вне зоны повтора снимает её);
  - двойной клик — поставить маркер;
  - зажать и потянуть — выделить зону A-B;
  - потянуть за границу зоны — подвинуть её (курсор меняется на ↔).
- **Маркер (⚑)** — одна точка на трек: отметь место, с которого продолжаешь работу над песней. «Играть с маркера» всегда начнёт оттуда (а без маркера — с начала). Маркер запоминается для каждого файла.
- **Зона повтора (A-B)** — выделенный на волне фрагмент играет по кругу; начало зоны автоматически становится маркером, так что «Играть с маркера» перезапускает зону с начала. Перемотка вручную за пределы зоны не затягивает обратно — луп срабатывает, только когда воспроизведение само дошло до границы B. Зона запоминается для каждого файла; снимается кликом вне зоны или командой Clear A-B loop zone (маркер при этом остаётся).
- **Несколько аудио в заметке** — выпадающий список треков заметки.
- **Счётчик проигрываний и наигранное время** — рядом с названием трека (`▶ 27 · 2ч40м`). Прогон засчитывается, только когда реально наиграно 5 секунд (запустил и сразу остановил — не считается; пауза с продолжением накопление не сбрасывает). Каждый круг зоны повтора — новый прогон. Наигранное время копит только реальное звучание. Правый клик по счётчику — сброс обоих.
- **К заметке трека** — клик по названию трека (или `Ctrl+Alt+S`) возвращает к заметке, из которой подхвачен трек: если она открыта во вкладке — переключение туда, иначе открытие; если источник неизвестен — ищется любая заметка со ссылкой на этот файл.
- **Выгрузить трек** — кнопка ⏏ в строке трека (или команда `Unload track`): плеер очищается и ничего не держит, пока занимаешься другими делами; хоткеи не сработают случайно. Вернуть трек — кнопкой «Взять аудио из заметки» или просто открыв заметку с аудио.
- **Открыть внешне** (перенесено из плагина Open Audio Externally): кнопка ⧉ справа от названия трека в панели и такая же у каждого встроенного аудиоплеера в заметках — клик открывает файл в приложении по умолчанию, правый клик показывает в проводнике. Команды: `Open track in default app`, `Reveal track in system explorer`. Кнопки у встроенных плееров отключаются в настройках.

Маркер и зона видны прямо на волне (оранжевый флажок и подсвеченный фрагмент) — отдельных индикаторов в панели нет. Время, кнопки и громкость — на одной строке под волной.

## Горячие клавиши (по умолчанию)

Все команды переназначаются в Settings → Hotkeys. Русская раскладка продублирована.

| Команда | Клавиши |
| --- | --- |
| Play from marker (or from start) | `Ctrl+Alt+X` |
| Stop (двойное: следующий `Ctrl+Alt+X` сыграет с начала, маркер цел; тройное: удаляются маркер и зона) | `Ctrl+Alt+C` |
| Set marker at current position | `Ctrl+Alt+A` |
| Play/Pause | `Ctrl+Alt+P` |
| Open track's note | `Ctrl+Alt+S` |
| Seek back / forward | `Ctrl+Alt+,` / `Ctrl+Alt+.` |

Без дефолтных клавиш: Clear marker, Clear A-B loop zone, Load audio from current note, Open player panel, Unload track, Open track in default app, Reveal track in system explorer.

## Настройки

Помимо основных (режим подхвата, шаг перемотки, старт с маркера) есть «тонкая настройка»: порог зачёта прогона, окно двойного стопа, высота волны, кнопка у встроенных плееров.

## Данные

Маркеры, зоны повтора и счётчики хранятся в `data.json` плагина по пути аудиофайла; при переименовании файла в хранилище данные переносятся автоматически.

## Сборка

```bash
npm install
npm run build   # tsc typecheck + esbuild → main.js
npm run dev     # watch-режим
```
